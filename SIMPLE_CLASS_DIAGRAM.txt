## How to Use This Diagram

1. **Understanding Structure**: 
   - Top section shows Django models (database tables)
   - Middle section shows utility classes (business logic)
   - Bottom sections show relationships, data flows, and storage

2. **Reading Relationships**:
   - `1:1` = One-to-One relationship
   - `1:N` = One-to-Many relationship
   - `N:1` = Many-to-One relationship
   - `FK` = Foreign Key
   - `PK` = Primary Key

3. **Method Naming**:
   - `+` = Public method
   - `-` = Private method (internal use)
   - `()` = Method (function)
   - `:` = Type annotation

4. **Data Types**:
   - Integer, String, Text, Boolean, DateTime, JSON, IP, etc.
